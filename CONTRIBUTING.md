# Contributing

Contributions are welcome once the public seed settles. Please keep early changes small, reviewable, and aligned with the local-first reference implementation.

## Developer Certificate of Origin

This project starts with DCO-style contribution tracking. By contributing, you certify that you wrote the contribution or otherwise have the right to submit it under the repository licenses.

Add a sign-off line to commits:

```text
Signed-off-by: Your Name <you@example.com>
```

## Public Content Rules

- Do not submit private/customer checklist packs.
- Do not submit generated databases, compiled binaries, local logs, or release archives.
- Use generic examples unless a real-world asset has been explicitly reviewed and approved for public release.
- Keep dependencies portable and source-buildable through CMake.
